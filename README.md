## 📌 项目简介

这是一个基于 **Node.js** 的 **TRON（波场链）钱包地址生成与比对脚本**。

原理：

- 不断随机生成 **助记词**
- 通过 BIP39 + HD Wallet 派生 **TRON 私钥**
- 计算对应的 **TRON 地址**
- 与用户预先提供的 **目标地址列表**进行比对
- 若命中，则记录 **地址 + 助记词 + 私钥**

---

## ✨ 功能特性

- ✅ 支持 **无限数量目标地址**
- ✅ 实时生成地址并写入文件
- ✅ 每个地址都包含对应 **助记词**
- ✅ 命中目标地址立即记录
- ✅ 使用 `Set` 结构，高效 O(1) 查询
- ✅ 代码结构清晰，易于二次开发

---

## 📂 文件说明

| 文件名 | 说明 |
|--------|------|
| `index.js` | 主执行脚本 |
| `targets.txt` | 目标地址列表（每行一个地址） |
| `addresses.txt` | 已生成的地址 + 助记词（实时写入） |
| `matched.txt` | 命中目标地址的结果 |

---

## 🧠 工作原理简述

助记词 → Seed → HD Wallet → 私钥 → 公钥 → Keccak256 → TRON 地址（Base58Check）

## ⚙️ 使用方法

1. 克隆仓库：
```
git clone https://github.com/AungPhyoOo1931/Tron-Chain-Address-Collider.git
cd Tron-Chain-Address-Collider
```

2. 安装依赖：
```
npm install
```

3. 设置目标地址：
在 `targets.txt` 中写入 TRON 地址，每行一个，例如：
```
TQ1xxxxxxxxxxxxxxxxxxxx
TD2xxxxxxxxxxxxxxxxxxxx
TA3xxxxxxxxxxxxxxxxxxxx
```

4. 启动脚本：
```
node index.js
```

5. 修改生成数量：
在 `index.js` 中修改：
```javascript
const COUNT = 10000; // 单次运行生成地址数量
```
例如：
- `10000` → 生成 1 万次后自动停止  
- `100000` → 生成 10 万次后自动停止

## 📈 性能说明

- 目标地址判断：O(1)（使用 Set）  
- 性能瓶颈主要在助记词生成、PBKDF2（BIP39）、椭圆曲线运算  
- 目标地址数量对性能影响极小

## ❓ 常见问题

**Q：生成的地址是真的吗？**  
是的，地址生成过程完全符合 TRON 地址规范，可正常接收 TRX。

**Q：目标地址可以放多少个？**  
理论上无限，1 万 / 10 万都没有问题。

**Q：为什么 `addresses.txt` 会越来越大？**  
这是正常现象，建议：
- 定期清理  
- 或使用日志轮转策略
